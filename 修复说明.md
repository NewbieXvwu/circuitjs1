# 语法错误修复说明

## 问题描述

GitHub Actions工作流文件 `.github/workflows/build-tauri-windows.yml` 的第169行出现YAML语法错误。

### 错误原因

在YAML文件中使用PowerShell的双引号here-string (`@"..."@`) 会导致YAML解析器混淆，因为：
1. YAML使用引号来定义字符串
2. PowerShell的here-string使用 `@"` 开始
3. 这两种语法冲突导致YAML解析失败

### 原始代码（错误）

```yaml
# Create a README for portable version
@"
CircuitJS1 Portable Version
============================
...
"@ | Out-File -FilePath "$portableDir/README.txt" -Encoding UTF8
```

## 解决方案

将PowerShell的双引号here-string改为单引号here-string，并使用变量存储。

### 修复后的代码（正确）

```yaml
# Create a README for portable version
$readmeContent = @'
CircuitJS1 Portable Version
============================
...
'@
$readmeContent | Out-File -FilePath "$portableDir/README.txt" -Encoding UTF8
```

## 技术细节

### PowerShell Here-String 类型

1. **双引号 here-string** (`@"..."@`)
   - 支持变量展开
   - 支持转义序列
   - 在YAML中容易引起解析冲突

2. **单引号 here-string** (`@'...'@`)
   - 不支持变量展开（字面量）
   - 不处理转义序列
   - 在YAML中更安全
   - 适合纯文本内容

### 为什么这样修复有效

1. 单引号here-string (`@'...'@`) 不包含需要YAML特殊处理的字符
2. 使用变量存储内容，然后通过管道传递，代码结构更清晰
3. 保持了PowerShell语法的正确性
4. 符合YAML的多行字符串规范
